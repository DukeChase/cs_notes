# 案例分析专题

## 1 案例分析概述

### 1.1 考试形式

**案例分析题**：下午考试，共3道题，每题25分，总分75分，考试时间90分钟。

### 1.2 考查内容

1. **系统规划与可行性分析**
2. **需求工程**
3. **系统设计**
4. **软件架构设计**
5. **系统实施与测试**
6. **系统运行与维护**
7. **新技术应用**

## 2 案例分析方法

### 2.1 解题步骤

1. **审题**：仔细阅读案例背景和问题
2. **分析**：提取关键信息，理解问题要点
3. **构思**：运用知识点，构建答案框架
4. **作答**：条理清晰地书写答案
5. **检查**：检查答案完整性和准确性

### 2.2 审题技巧

1. **标记关键词**：圈出案例中的关键信息
2. **理解问题要求**：明确题目问什么
3. **关联知识点**：将问题与知识点对应

### 2.3 作答技巧

1. **分点作答**：使用序号或要点形式
2. **逻辑清晰**：答案要有逻辑性
3. **专业术语**：使用专业术语
4. **简洁明了**：避免冗长
5. **结合案例**：答案要结合案例内容

## 3 常见题型分析

### 3.1 概念理解题

**特点**：考查对基本概念的理解

**解题方法**：
- 准确理解概念定义
- 结合案例说明
- 注意区分相似概念

### 3.2 方法应用题

**特点**：要求运用某种方法分析问题

**解题方法**：
- 掌握方法的步骤和要点
- 按步骤分析
- 结合案例实际

### 3.3 比较分析题

**特点**：比较两种或多种方法的异同

**解题方法**：
- 列出比较维度
- 逐一对比
- 说明适用场景

### 3.4 设计题

**特点**：要求设计系统架构或方案

**解题方法**：
- 明确设计目标
- 选择合适的方法或模式
- 说明设计理由
- 考虑优缺点

## 4 知识点梳理

### 4.1 系统规划与可行性

- 可行性分析类型
- 成本效益分析方法
- NPV、ROI、投资回收期计算

### 4.2 需求工程

- 需求获取方法
- UML建模（用例图、类图、序列图等）
- 需求分析方法

### 4.3 系统设计

- 结构化设计方法
- 面向对象设计方法
- 设计模式应用
- 内聚与耦合

### 4.4 软件架构

- 架构风格选择
- 架构评估方法
- 质量属性分析

### 4.5 项目管理

- 进度管理（关键路径、时差计算）
- 成本管理（挣值分析）
- 风险管理
- 配置管理

## 5 答题注意事项

### 5.1 常见错误

1. **答非所问**：没有理解问题要求
2. **概念混淆**：混淆相似概念
3. **遗漏要点**：答案不完整
4. **逻辑混乱**：答案缺乏逻辑性
5. **脱离案例**：答案没有结合案例

### 5.2 评分要点

1. **准确性**：答案正确
2. **完整性**：答案全面
3. **逻辑性**：条理清晰
4. **专业性**：使用专业术语
5. **结合案例**：理论联系实际
