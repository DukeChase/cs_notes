# 企业信息化

## 1. 企业信息化概述

### 1.1 信息化基本概念

**信息化定义**：是指培养、发展以计算机为主的智能化工具为代表的新生产力，并使之造福于社会的历史过程。

**企业信息化定义**：是指企业在作业、管理、经营等各个层次、各个环节和各个领域，采用现代信息技术，特别是网络技术，充分开发和广泛利用企业内外信息资源，不断提高生产、经营、管理、决策的效率和水平，逐步实现企业运行的全面自动化，进而提高企业经济效益和企业竞争力的过程。

**企业信息化的目的**：
- 提高企业经济效益
- 提高企业竞争力
- 实现企业运行的全面自动化

### 1.2 企业信息化的层次

| 层次 | 说明 | 典型应用 |
|------|------|---------|
| **产品信息化** | 产品中融入信息技术 | 智能产品、数字产品 |
| **设计信息化** | 产品设计过程的信息化 | CAD、CAM、CAE |
| **生产过程信息化** | 生产制造过程的信息化 | MES、DCS、PLC |
| **企业管理信息化** | 企业管理过程的信息化 | ERP、CRM、SCM、OA |
| **经营决策信息化** | 企业决策过程的信息化 | BI、DSS、数据挖掘 |

### 1.3 企业信息化的内容

**企业信息化的主要内容**：

1. **生产制造信息化**
   - 计算机辅助设计（CAD）
   - 计算机辅助制造（CAM）
   - 计算机辅助工程（CAE）
   - 产品数据管理（PDM）
   - 制造执行系统（MES）

2. **企业管理信息化**
   - 企业资源计划（ERP）
   - 客户关系管理（CRM）
   - 供应链管理（SCM）
   - 人力资源管理（HRM）
   - 财务管理
   - 办公自动化（OA）

3. **经营决策信息化**
   - 商业智能（BI）
   - 决策支持系统（DSS）
   - 数据挖掘
   - 大数据分析

### 1.4 企业信息化的发展阶段

**诺兰模型（Nolan Model）**：

| 阶段 | 名称 | 特征 |
|------|------|------|
| 第一阶段 | 初始期 | 引入计算机，初步应用 |
| 第二阶段 | 普及期 | 应用扩散，系统增多 |
| 第三阶段 | 控制期 | 开始控制，减少冗余 |
| 第四阶段 | 整合期 | 系统集成，数据共享 |
| 第五阶段 | 数据管理期 | 数据成为战略资源 |
| 第六阶段 | 成熟期 | 信息系统与业务完全融合 |

### 1.5 企业信息化的建设原则

1. **统一规划，分步实施**
2. **需求导向，突出重点**
3. **先进适用，量力而行**
4. **资源共享，互联互通**
5. **安全可靠，注重实效**

## 2. 企业资源计划（ERP）

### 2.1 ERP概述

**ERP定义**：**Enterprise Resource Planning**，企业资源计划，是指建立在信息技术基础上，以系统化的管理思想，为企业决策层及员工提供决策运行手段的管理平台。

**ERP的核心思想**：
- 对整个供应链资源进行管理
- 精益生产、同步工程和敏捷制造
- 事先计划与事中控制

**ERP的发展历程**：

| 阶段 | 名称 | 核心功能 |
|------|------|---------|
| 1960s | 库存控制订货点法 | 库存管理 |
| 1970s | 物料需求计划（MRP） | 物料需求计算 |
| 1980s | 闭环MRP | 增加能力需求计划 |
| 1980s | 制造资源计划（MRP II） | 扩展到生产、财务、销售 |
| 1990s | 企业资源计划（ERP） | 集成供应链管理 |
| 2000s | ERP II | 扩展到电子商务、协同商务 |

### 2.2 ERP的主要功能模块

**ERP系统的主要功能模块**：

| 模块 | 功能说明 |
|------|---------|
| **财务管理** | 总账、应收应付、固定资产、成本管理、预算管理 |
| **供应链管理** | 采购管理、销售管理、库存管理、物流管理 |
| **生产管理** | 生产计划、物料需求、车间管理、质量管理 |
| **人力资源管理** | 人事管理、薪酬管理、考勤管理、绩效管理 |
| **客户关系管理** | 客户信息管理、销售自动化、服务管理 |
| **项目管理** | 项目计划、项目执行、项目监控 |

### 2.3 ERP的实施

**ERP实施的关键因素**：

| 因素       | 说明           |
| -------- | ------------ |
| **高层支持** | 企业高层领导的重视和支持 |
| **明确目标** | 清晰的实施目标和范围   |
| **流程优化** | 业务流程重组（BPR）  |
| **数据准备** | 准确、完整的基础数据   |
| **人员培训** | 充分的用户培训      |
| **项目管理** | 专业的项目管理      |
| **持续改进** | 实施后的持续优化     |

**ERP实施的生命周期**：

1. **项目准备**：立项、组建团队、制定计划
2. **蓝图设计**：需求分析、流程设计、方案制定
3. **系统实现**：系统配置、开发、数据准备
4. **上线准备**：用户培训、模拟测试、数据迁移
5. **系统上线**：切换上线、运行支持
6. **持续支持**：运维优化、持续改进

**ERP实施的风险**：
- 需求不明确
- 业务流程重组困难
- 数据质量问题
- 用户抵触
- 项目延期
- 成本超支

## 3. 客户关系管理（CRM）

### 3.1 CRM概述

**CRM定义**：**Customer Relationship Management**，客户关系管理，是指企业利用信息技术和互联网技术，协调企业与顾客间在销售、营销和服务上的交互，从而提升其管理方式，向客户提供创新式的个性化服务的过程。

**CRM的核心思想**：
- 以客户为中心
- 提高客户满意度
- 提高客户忠诚度
- 提升企业竞争力

**CRM的价值**：
- 提高销售收入
- 降低销售成本
- 提高客户满意度
- 提高客户保留率

### 3.2 CRM的功能

**CRM的主要功能模块**：

| 模块             | 功能说明                     |
| -------------- | ------------------------ |
| **销售自动化（SFA）** | 线索管理、商机管理、报价管理、订单管理、销售预测 |
| **营销自动化**      | 营销活动管理、线索培育、邮件营销、市场分析    |
| **客户服务**       | 服务请求管理、工单管理、知识库、呼叫中心     |
| **客户数据管理**     | 客户信息管理、联系人管理、交互历史        |
| **分析报表**       | 销售分析、客户分析、营销分析           |

### 3.3 CRM的类型

| 类型 | 说明 | 特点 |
|------|------|------|
| **运营型CRM** | 面向日常业务操作 | 销售自动化、服务自动化、营销自动化 |
| **分析型CRM** | 面向数据分析 | 数据仓库、数据挖掘、客户分析 |
| **协作型CRM** | 面向客户交互 | 呼叫中心、电子邮件、社交媒体 |

### 3.4 CRM与ERP的关系

**CRM与ERP的集成**：
- CRM侧重于前端，管理客户关系
- ERP侧重于后端，管理企业资源
- 两者集成实现从前端到后端的全流程管理

**集成内容**：
- 客户信息共享
- 订单信息同步
- 库存信息查询
- 财务信息对接

## 4. 供应链管理（SCM）

### 4.1 SCM概述

**SCM定义**：**Supply Chain Management**，供应链管理，是指围绕核心企业，通过对信息流、物流、资金流的控制，从采购原材料开始，制成中间产品及最终产品，最后由销售网络把产品送到消费者手中的整体功能网链结构模式。

**供应链的组成**：

| 环节      | 说明        |
| ------- | --------- |
| **供应商** | 提供原材料和零部件 |
| **制造商** | 生产加工产品    |
| **分销商** | 产品分销      |
| **零售商** | 产品销售      |
| **消费者** | 最终用户      |

**SCM的目标**：
- 降低供应链成本
- 提高供应链效率
- 提高客户服务水平
- 提高供应链的响应速度

### 4.2 SCM的功能

**SCM的主要功能模块**：

| 模块 | 功能说明 |
|------|---------|
| **需求管理** | 需求预测、需求计划 |
| **采购管理** | 供应商管理、采购计划、采购执行 |
| **生产计划** | 主生产计划、物料需求计划 |
| **库存管理** | 库存控制、仓库管理 |
| **物流管理** | 运输管理、配送管理 |
| **协同计划** | 协同规划、预测与补货（CPFR） |

### 4.3 供应链协同

**CPFR（协同规划、预测与补货）**：
- 供应链上下游企业协同制定计划
- 共享预测信息
- 协同补货决策

**VMI（供应商管理库存）**：
- 由供应商管理客户的库存
- 根据客户需求自动补货
- 降低库存成本

**JIT（准时制生产）**：
- 在需要的时间，按需要的量，生产所需的产品
- 降低库存，提高生产效率

## 5. 电子商务

### 5.1 电子商务概述

**电子商务定义**：是指在全球各地广泛的商业贸易活动中，在因特网开放的网络环境下，基于浏览器/服务器应用方式，买卖双方不谋面地进行各种商贸活动，实现消费者的网上购物、商户之间的网上交易和在线电子支付以及各种商务活动、交易活动、金融活动和相关的综合服务活动的一种新型的商业运营模式。

**电子商务的特点**：
- 全球性
- 便捷性
- 低成本
- 高效率
- 个性化

### 5.2 电子商务的类型

| 类型 | 名称 | 说明 | 示例 |
|------|------|------|------|
| **B2B** | 企业对企业 | 企业间的电子商务 | 阿里巴巴 |
| **B2C** | 企业对消费者 | 企业向消费者销售 | 京东、天猫 |
| **C2C** | 消费者对消费者 | 消费者之间的交易 | 淘宝、eBay |
| **C2B** | 消费者对企业 | 消费者定制，企业生产 | 团购、定制 |
| **O2O** | 线上到线下 | 线上引流，线下消费 | 美团、大众点评 |

### 5.3 电子商务系统组成

**电子商务系统的基本组成**：

| 组成部分       | 说明                         |
| ---------- | -------------------------- |
| **网络平台**   | Internet、Intranet、Extranet |
| **基础服务平台** | 支付平台、物流平台、安全认证             |
| **应用系统**   | 网上商店、电子采购、电子市场             |
| **安全保障**   | 防火墙、加密、数字证书                |
| **法律法规**   | 电子签名法、电子商务法                |

### 5.4 电子支付

**电子支付方式**：

| 方式 | 说明 |
|------|------|
| **网上银行** | 通过银行网站进行支付 |
| **第三方支付** | 支付宝、微信支付 |
| **移动支付** | 手机支付、NFC支付 |
| **数字货币** | 比特币、央行数字货币 |

**电子支付安全**：
- SSL/TLS加密
- 数字证书
- 短信验证
- 动态口令
- 生物识别

### 5.5 电子商务物流

**电子商务物流模式**：

| 模式 | 说明 | 示例 |
|------|------|------|
| **自建物流** | 企业自建物流体系 | 京东物流 |
| **第三方物流** | 委托专业物流公司 | 顺丰、圆通 |
| **物流联盟** | 多家企业共建物流 | 菜鸟网络 |
| **众包物流** | 利用社会运力 | 达达、闪送 |

## 6. 电子政务

### 6.1 电子政务概述

**电子政务定义**：是指政府机构运用现代计算机和网络技术，将其管理和服务职能转移到网络上去完成，同时实现政府组织结构和工作流程的重组优化，超越时间、空间和部门分隔的制约，向社会提供优质、规范、透明、符合国际水准的管理和服务。

**电子政务的意义**：
- 提高政府工作效率
- 降低行政成本
- 增强政府透明度
- 改善公共服务
- 促进民主参与

### 6.2 电子政务的类型

| 类型 | 名称 | 说明 |
|------|------|------|
| **G2G** | 政府对政府 | 政府内部、政府之间的电子政务 |
| **G2B** | 政府对企业 | 政府向企业提供的服务 |
| **G2C** | 政府对公民 | 政府向公民提供的服务 |
| **G2E** | 政府对公务员 | 政府内部公务员管理 |

### 6.3 电子政务系统

**电子政务的主要应用**：

| 应用领域 | 说明 |
|---------|------|
| **办公自动化** | 公文流转、会议管理、日程管理 |
| **行政审批** | 在线申请、审批流程、结果查询 |
| **公共服务** | 社保查询、税务申报、交通违章查询 |
| **信息公开** | 政策法规、政府公告、统计数据 |
| **互动交流** | 领导信箱、在线咨询、民意调查 |

### 6.4 电子政务的发展阶段

| 阶段 | 名称 | 特征 |
|------|------|------|
| 第一阶段 | 起步阶段 | 政府上网，信息发布 |
| 第二阶段 | 单向服务阶段 | 提供单向服务，信息查询 |
| 第三阶段 | 双向互动阶段 | 在线交互，表单提交 |
| 第四阶段 | 在线事务处理阶段 | 完整的在线事务处理 |
| 第五阶段 | 无缝集成阶段 | 跨部门整合，一站式服务 |

### 6.5 "互联网+政务服务"

**"互联网+政务服务"的目标**：
- 让数据多跑路，让群众少跑腿
- 实现"一网通办"
- 推进政务服务标准化、精准化、便捷化

**主要措施**：
- 统一身份认证
- 统一电子证照
- 统一数据共享
- 统一支付平台

## 7. 企业应用集成（EAI）

### 7.1 EAI概述

**EAI定义**：Enterprise Application Integration，企业应用集成，是指将企业内部不同的、相互之间相对独立的软件系统集成起来，实现数据共享和业务流程自动化。

**EAI的必要性**：
- 企业信息系统孤岛问题
- 数据不一致问题
- 业务流程断裂问题
- 信息重复录入问题

**EAI的目标**：
- 实现数据共享
- 实现业务流程集成
- 提高企业运营效率
- 降低IT维护成本

### 7.2 EAI的层次

| 层次 | 名称 | 说明 |
|------|------|------|
| **数据集成** | 数据层集成 | 实现数据的共享和同步 |
| **应用集成** | 应用层集成 | 实现应用系统之间的互操作 |
| **业务流程集成** | 过程层集成 | 实现跨系统的业务流程 |
| **表示集成** | 界面层集成 | 实现统一的用户界面 |

### 7.3 EAI的技术

**EAI的主要技术**：

| 技术        | 说明                  |
| --------- | ------------------- |
| **数据库访问** | ODBC、JDBC、数据库网关     |
| **消息中间件** | MQ、JMS、消息队列         |
| **应用服务器** | J2EE、.NET           |
| **Web服务** | SOAP、REST、WSDL、UDDI |
| **ESB**   | 企业服务总线              |
| **ETL**   | 数据抽取、转换、加载          |

### 7.4 ESB（企业服务总线）

**ESB定义**：Enterprise Service Bus，企业服务总线，是一种软件架构模式，用于设计和实现面向服务架构（SOA）中应用程序之间的交互通信。

**ESB的功能**：
- 服务路由
- 协议转换
- 消息转换
- 服务编排
- 安全控制
- 监控管理

**ESB的优势**：
- 降低系统耦合度
- 提高系统灵活性
- 简化系统维护
- 支持服务复用

## 8. 商业智能（BI）

### 8.1 BI概述

**BI定义**：Business Intelligence，商业智能，是指利用数据仓库、数据挖掘等技术对商业数据进行分析和处理，为管理者的决策过程提供支持。

**BI的核心能力**：
- 数据整合
- 数据分析
- 数据可视化
- 决策支持

**BI的价值**：
- 提高决策质量
- 发现商业机会
- 优化业务流程
- 提升竞争优势

### 8.2 BI的组成

**BI系统的主要组成部分**：

| 组成部分      | 说明                 |
| --------- | ------------------ |
| **数据源**   | 业务系统、外部数据、手工数据     |
| **ETL**   | 数据抽取、转换、加载         |
| **数据仓库**  | 面向主题的、集成的、非易失的数据集合 |
| **数据集市**  | 面向特定部门或应用的数据集合     |
| **OLAP**  | 联机分析处理，多维数据分析      |
| **数据挖掘**  | 从大量数据中发现模式和规律      |
| **报表工具**  | 固定报表、即席查询          |
| **可视化工具** | 仪表盘、图表、地图          |
### 8.3 数据仓库

**数据仓库特点**：
- 面向主题
- 集成性
- 非易失性
- 时变性

**数据仓库架构**：

| 层次      | 说明           |
| ------- | ------------ |
| **ODS** | 操作数据存储，近实时数据 |
| **DW**  | 数据仓库，历史数据    |
| **DM**  | 数据集市，主题数据    |
### 8.4 OLAP

**OLAP定义**：Online Analytical Processing，联机分析处理，是一种针对特定问题的联机数据访问和分析技术。

**OLAP的基本操作**：

| 操作 | 说明 |
|------|------|
| **上卷（Roll-up）** | 沿维度向上汇总 |
| **下钻（Drill-down）** | 沿维度向下细分 |
| **切片（Slice）** | 固定一个维度，分析其他维度 |
| **切块（Dice）** | 固定多个维度，分析其他维度 |
| **旋转（Pivot）** | 改变维度的显示方向 |

**OLAP的类型**：

| 类型 | 名称 | 特点 |
|------|------|------|
| **ROLAP** | 关系型OLAP | 基于关系数据库 |
| **MOLAP** | 多维OLAP | 基于多维数组 |
| **HOLAP** | 混合OLAP | 结合ROLAP和MOLAP |

### 8.5 数据挖掘

**数据挖掘定义**：是从大量的、不完全的、有噪声的、模糊的、随机的数据中提取隐含在其中的、人们事先不知道的、但又是潜在有用的信息和知识的过程。

**数据挖掘的任务**：

| 任务 | 说明 |
|------|------|
| **分类** | 将数据映射到预定义的类别 |
| **聚类** | 将数据分成不同的组 |
| **关联分析** | 发现数据项之间的关联规则 |
| **预测** | 基于历史数据预测未来趋势 |
| **异常检测** | 发现数据中的异常点 |

**数据挖掘的算法**：
- 决策树
- 神经网络
- 支持向量机
- 关联规则（Apriori）
- 聚类算法（K-means）

## 10. 信息化规划与实施

**信息化评价的维度**：

| 维度 | 说明 |
|------|------|
| **投入评价** | 资金投入、人员投入、时间投入 |
| **过程评价** | 项目管理、实施质量、风险控制 |

### 10.3 信息化与业务流程重组

**BPR定义**：Business Process Reengineering，业务流程重组，是指对企业业务流程进行根本性的再思考和彻底性的再设计，从而在成本、质量、服务和速度等方面获得显著改善。

**信息化与BPR的关系**：
- 信息化是BPR的使能器
- BPR是信息化成功的前提
- 两者相辅相成

**BPR的原则**：
- 以流程为中心
- 以客户为导向
- 打破部门壁垒
- 利用信息技术

## 11. 考试真题精选

### 真题1：ERP发展历程
ERP系统的发展经历了MRP、闭环MRP、（**MRP II**）和ERP四个阶段。

### 真题2：CRM核心
CRM的核心思想是以（**客户**）为中心。

### 真题3：电子商务类型
企业与消费者之间的电子商务称为（**B2C**），企业与企业之间的电子商务称为（**B2B**）。

### 真题4：电子政务类型
政府对公民的电子政务称为（**G2C**），政府对企业的电子政务称为（**G2B**）。

### 真题5：EAI层次
企业应用集成包括数据集成、（**应用集成**）、业务流程集成和表示集成四个层次。

### 真题6：BI组成
商业智能系统通常包括数据仓库、（**ETL**）、OLAP和数据挖掘等组成部分。

### 真题7：数据仓库特点
数据仓库的特点包括面向主题、（**集成性**）、非易失性和时变性。

### 真题8：OLAP操作
OLAP的基本操作包括上卷、下钻、（**切片**）、切块和旋转。

### 真题9：SCM目标
供应链管理的目标是在满足客户需要的前提下，对整个供应链的各个环节进行综合管理，实现（**降低供应链成本**）、提高供应链效率的目标。

### 真题10：诺兰模型
诺兰模型将企业信息化发展分为初始期、普及期、控制期、（**整合期**）、数据管理期和成熟期六个阶段。
