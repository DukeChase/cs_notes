## 2 移动应用架构

### 2.1 原生应用

**优点**：
- 性能最佳
- 用户体验好
- 可充分利用设备功能

**缺点**：
- 开发成本高
- 需要多个开发团队
- 维护成本高

### 2.2 混合应用

**优点**：
- 开发成本低
- 跨平台
- 维护简单

**缺点**：
- 性能较差
- 用户体验不如原生

**技术栈**：
- Ionic + Angular
- React Native
- Flutter

### 2.3 跨平台应用

**框架对比**：

| 框架 | 语言 | 特点 |
|------|------|------|
| **React Native** | JavaScript | Facebook推出，生态丰富 |
| **Flutter** | Dart | Google推出，性能好 |
| **Xamarin** | C# | Microsoft推出，.NET生态 |
