# 系统设计

## 1 系统设计概述

### 1.1 系统设计定义

**系统设计**：将需求转换为系统设计方案的过程，包括总体设计和详细设计两个阶段。

### 1.2 系统设计原则

1. **模块化**：将系统分解为独立的模块
2. **抽象**：隐藏实现细节
3. **信息隐藏**：模块内部信息对外不可见
4. **高内聚低耦合**

## 2 结构化设计方法

### 2.1 结构图

**结构图**：描述系统模块层次结构和调用关系的图形工具。

**基本符号**：
- 模块
- 调用
- 数据
- 控制信息

### 2.2 内聚类型

| 内聚类型 | 强度 | 说明 |
|---------|------|------|
| **偶然内聚** | 最弱 | 模块内的元素之间没有关联 |
| **逻辑内聚** | 弱 | 逻辑上相关的功能组合在一起 |
| **时间内聚** | 弱 | 同一时间执行的功能组合在一起 |
| **过程内聚** | 中 | 按过程顺序组合的功能 |
| **通信内聚** | 中 | 使用相同数据的功能组合在一起 |
| **顺序内聚** | 强 | 一个功能的输出是另一个功能的输入 |
| **功能内聚** | 最强 | 模块内所有元素共同完成一个功能 |

### 2.3 耦合类型

| 耦合类型 | 强度 | 说明 |
|---------|------|------|
| **内容耦合** | 最强 | 一个模块直接访问另一个模块的内部数据 |
| **公共耦合** | 强 | 多个模块访问同一个公共数据环境 |
| **外部耦合** | 强 | 模块间通过外部环境关联 |
| **控制耦合** | 中 | 一个模块控制另一个模块的执行逻辑 |
| **标记耦合** | 中 | 模块间通过数据结构传递信息 |
| **数据耦合** | 弱 | 模块间通过数据参数传递信息 |
| **非直接耦合** | 最弱 | 模块间没有直接关系 |

## 3 面向对象设计方法

### 3.1 面向对象基本概念

- **类**：对象的模板
- **对象**：类的实例
- **封装**：隐藏对象的内部细节
- **继承**：子类继承父类的属性和方法
- **多态**：同一操作作用于不同对象产生不同结果

### 3.2 设计原则（SOLID）

| 原则 | 名称 | 说明 |
|------|------|------|
| **S** | 单一职责原则 | 一个类只负责一个职责 |
| **O** | 开闭原则 | 对扩展开放，对修改关闭 |
| **L** | 里氏替换原则 | 子类可以替换父类 |
| **I** | 接口隔离原则 | 使用多个隔离的接口 |
| **D** | 依赖倒置原则 | 依赖抽象，不依赖具体 |

## 4 设计模式

### 4.1 创建型模式

| 模式 | 说明 |
|------|------|
| **单例模式** | 确保一个类只有一个实例 |
| **工厂方法模式** | 定义创建对象的接口 |
| **抽象工厂模式** | 创建一系列相关对象 |
| **建造者模式** | 分步骤创建复杂对象 |
| **原型模式** | 通过复制创建对象 |

### 4.2 结构型模式

| 模式 | 说明 |
|------|------|
| **适配器模式** | 将接口转换为客户期望的接口 |
| **桥接模式** | 将抽象部分与实现部分分离 |
| **组合模式** | 将对象组合成树形结构 |
| **装饰器模式** | 动态地给对象添加职责 |
| **外观模式** | 为子系统提供统一接口 |
| **享元模式** | 共享细粒度对象 |
| **代理模式** | 控制对对象的访问 |

### 4.3 行为型模式

| 模式 | 说明 |
|------|------|
| **策略模式** | 定义一系列算法 |
| **模板方法模式** | 定义算法的骨架 |
| **观察者模式** | 对象间一对多的依赖关系 |
| **迭代器模式** | 顺序访问集合元素 |
| **责任链模式** | 将请求沿链传递 |
| **命令模式** | 将请求封装为对象 |
| **备忘录模式** | 保存对象的状态 |
| **状态模式** | 允许对象在状态改变时改变行为 |
| **访问者模式** | 操作复杂对象结构 |
| **中介者模式** | 用中介对象封装对象间的交互 |
| **解释器模式** | 定义语言的语法和解释器 |
